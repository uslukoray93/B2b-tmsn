Proje Tanımı

TMSN-B2B Gediz Makina, tarım makineleri sektöründe faaliyet gösteren bir firma için geliştirilmiş, bayiler arası işlem yapılmasını sağlayan bir B2B (Business-to-Business) e-ticaret platformudur.

Temel Özellikler

  • ✅ Bayi, Admin ve Müşteri Temsilcisi olmak üzere 3 ayrı kullanıcı rolü
  • ✅ Online sipariş yönetimi ve takibi
  • ✅ Kredi kartı ve vadeli ödeme entegrasyonu
  • ✅ Kredi limiti ve cari hesap yönetimi
  • ✅ Garanti, iade ve tamir talep sistemleri
  • ✅ Detaylı raporlama ve analiz araçları

  • ✅ Responsive (mobil uyumlu) tasarım

🔴 Kritik Güvenlik Açıkları

1. SQL Injection (ÇOK YÜKSEK RİSK)

Durum: Tüm projede yaygın - 100+ lokasyonda tespit edildi

Örnek Kod:

// ❌ KÖTÜ - SQL Injection açığı string sql = "SELECT * FROM Cariler WHERE CariKodu = '" + txtCariKodu.Value + "'"; cn.DataTableGetir(sql); // ✅ DOĞRU KULLANIM (Parametrize query) SqlCommand cmd = new SqlCommand("SELECT * FROM Cariler WHERE CariKodu = @CariKodu"); cmd.Parameters.AddWithValue("@CariKodu", txtCariKodu.Value);

Etki:

  • Veritabanı manipülasyonu
  • Tüm verilerin çalınması
  • Veri silme/değiştirme
  • Unauthorized access

2. MD5 Password Hashing (YÜKSEK RİSK)

// ❌ KÖTÜ - MD5 kullanımı string parola = FormsAuthentication.HashPasswordForStoringInConfigFile(txtParola.Value, "MD5"); // ✅ DOĞRU - BCrypt kullanımı using BCrypt.Net; string hashedPassword = BCrypt.HashPassword(password); bool isValid = BCrypt.Verify(password, hashedPassword);

Sorunlar:

  • MD5 kırılabilir (rainbow tables)
  • Salt kullanılmıyor
  • Tek yönlü hash değil

3. Hardcoded Credentials (YÜKSEK RİSK)

// Ticket.cs - Açık şifre string _ConnectionStrings = "Data Source=185.139.5.125; Initial Catalog=Ticket; User ID=sa; Password=Cs153575";

Riskler:

  • Kaynak kodunda açık şifre
  • SA (admin) hesabı kullanımı
  • Git history'de şifre görünür

4. SA Hesabı Kullanımı (ORTA-YÜKSEK RİSK)

SQL Server'a SA (System Administrator) hesabıyla bağlanılıyor.

Sorunlar:

  • Least privilege prensibi ihlali
  • Tüm veritabanına tam yetki
  • Audit trail yok

🟡 Orta Seviye Güvenlik Sorunları

5. Error Handling - Information Disclosure

catch (Exception ex) { // ❌ Detaylı hata mesajı kullanıcıya gösteriliyor lblUyari.InnerText = ex.Message.ToString(); throw new Exception(ex.Message); // Stack trace kayboldu }

6. Session Güvenliği

  • Session fixation koruması yok
  • HTTPS zorlaması belirsiz
  • Secure/HttpOnly cookie flags kontrol edilmeli

7. Input Validation Eksikliği

  • Client-side validation var (ASP.NET validators)
  • Server-side validation yetersiz
  • XSS koruması belirsiz

🟢 Pozitif Güvenlik Özellikleri

  • ✅ Session timeout mekanizması mevcut
  • ✅ ViewState kullanımı (tampering koruması)
  • ✅ Session keep-alive (kullanıcı deneyimi)
  • ✅ DevExpress kontrolleri (built-in encoding)

Acil Önlemler

📌 HEMEN YAPILMASI GEREKENLER:
  1. Tüm SQL sorgularını parametrize et (1-2 hafta)
  2. MD5'i BCrypt ile değiştir (1 hafta)
  3. Connection string'i şifrele (1 gün)
  4. SA hesabı yerine limited user oluştur (1 gün)
  5. HTTPS zorunlu hale getir (1 gün)
  6. Detaylı error logging ekle (3 gün)
